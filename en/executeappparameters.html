Executing applications with parameters
Use cases > Executing applications with parameters

When the Command associated with an Option is a document, its display is usually associated with a certain application installed on the PC. The association between a certain type of document and the application used to present it is made through the extension of the document file. E.g., files with extension ".doc" or ".docx" may be associated with the Word application, but can also be associated with the LibreOffice Write application. The default association is maintained in the Windows registry, and can be viewed, and changed, through Windows Explorer: click on the file with the right mouse button and look for "Open with": a menu will appear with all installed applications that can be used to open it.

Additionally, the concept Option Command allows you to choose the application that actually should be used to open a certain document, regardless of which application is associated with its extension.

For example, a document with extension PDF may be associated with the application "Adobe Reader", and yet we may want to open it with another one, say, Foxit Reader that allows for some editing (comments, underlining, etc.)

On the other hand, the document we want to use in the Option may not have an associated application.

The alternative to these situations is to set the Command of the Option as referring to the application (executable) of our choice, followed by the actual document as parameter. This can be done very easily with the "Change Command" option of the Options menu. See here.
