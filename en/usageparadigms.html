Usage paradigms

The Desktop Organizer can be used with two documentation organization paradigms.

Paradigm of the Librarian

When a book arrives at a library, the librarian has to do at least two things:

  1. Decide where it is going to be located (in which shelf).
  2. Make one or more descriptive cards that will be used by visitors to the library to locate it.

Action 1) may be conditioned by various physical factors, and also by the spirit of the library itself. For example, the location can be decided by the subject of the book, or type; and within this location alphabetically by surname of the author (or authors).

The creation of the descriptive cards may contemplate many other aspects, to facilitate the choice of the book by a potential interested visitor. The concepts here are:

  • Author or authors,
  • Title,
  • The theme or themes of the book,
  • The type of document (textbook, essay, novel, historical novel, treaty, international agreement, report, etc.)
  • The book publishing company,
  • The name of the particular edition of that book,
  • and many others.

Reader's paradigm

A reader invests some effort in reading a certain book or document, and as a result he/she take some decisions: for example, he/she may discard it (not liked it, or does not refer to something that interests him/her), or he/she may want to remember it as a reference, or may take as reference for a particular project. In all cases, he/she may decide to write a note about important aspects of his/her reading, at that time. A photo in page 23, a graphic in page 74, a phrase in page 15, etc. You may want to remember some aspects of it (the photo on page 23, the graphic on page 74, the theory set out in chapter 3, etc.)

The reader may want to have a list of readings on a particular topic, and the same document may be on more than one of these lists.

In all these cases, the reader would appreciate having a tool that facilitates:

  • taking notes on the document/book.
  • storing these notes in ways that ease their recovery.
  • associate the notes to the document.
  • retrieving the document taking into account the notes, or tags.

The Desk Organizer has been developed thinking about the needs reflected in these two paradigms: librarian and / or reader.

The starting scenario is a library (or set of documents in digital form), formed by a set of files of different types, located in a directory structure and subdirectories.

The user (librarian and / or reader) must create configuration files, as many as he wishes. The name given to a configuration file can be significant for a particular purpose: b.p., "doctoral thesis", or "History" or "Claim to the Bank", etc. The name given to a configuration file can be significant for a particular purpose: i.e., "doctoral thesis", or "History" or "Claim to the Bank", etc.

The program presents an initial View (Standard View) that corresponds to the concept of "List of topics", named Groups, within which there are references to documents or applications, named Options.

Groups have two properties "Title" and "Comment", and the program has several ways to create and edit them. One of these ways is by selecting a file and dragging it to the left panel (see here).

A reference to a document or application (Option within a Group) has the following  properties:

  • A Command.
  • A Comment.
  • A set of Tags (including none).

The Command may refers to:

    • the location of the file associated with the document, or
    • the location of an applications (executable file), with optional parameters, or
    • a URL.

The set of Tags may be hierarchical (nested). I.e. "novel", "novel:historic", "novel:historic:american". Tag are the tool to achieve classification of a document, regardless its location.

The program offers several ways to access the documents (Options): in the Standard View, the  Groups follows the "List of readings" concept.

The "Tag Tree View" ease the search of documents by exploring the tree of tags.

The "Search View" add the functionality of searching by a combination of free text and/or sets of tags.

The View by last execution date also helps find documents recently used.

In all the Views the program presents, immediately, the content of the "Comment" and the "Tags" associated with the Option on which the mouse is set.

Actual access a the document, URL or execution of the associated application is just a matter of double-clicking on it, regardless the View in use.

The Comment of an Option or a Group is editable at any time, through a form that is presented with a menu option, very accessible.

The set of Tags of an Option can also be changed very easy.

In the Desk Organizer, the priority is the immediacy of access to important information and the ease of making changes.
